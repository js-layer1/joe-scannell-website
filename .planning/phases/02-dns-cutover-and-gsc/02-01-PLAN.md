<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Add domain to Vercel, pre-generate SSL, cut DNS from Carrd</name>
  <action>
The user must perform the following steps in order. Claude will verify each step with CLI commands after the user reports completion.

**Step 1 -- Add domain to Vercel:**
1. Go to Vercel > Project Settings > Domains for the joe-scannell-website project
2. Add `joescannell.com` as a custom domain
3. When Vercel prompts to add `www.joescannell.com`, accept (configure as redirect to apex)
4. Note the IP address or DNS values Vercel displays (likely `76.76.21.21` for A record, `cname.vercel-dns.com` for www CNAME). Write these down.

**Step 2 -- Pre-generate SSL certificate:**
1. In Vercel domain settings, find "SSL Certificates" section
2. Click "Pre-generate SSL certificates" (or similar)
3. Vercel displays one or more TXT records to add to DNS
4. Go to Cloudflare > joescannell.com zone > DNS
5. Add the TXT record(s) Vercel requested
6. Return to Vercel and click "Verify" for SSL
7. Wait for Vercel to confirm the SSL certificate is issued

**Step 3 -- Cut DNS in Cloudflare:**
1. In Cloudflare DNS for joescannell.com:
   - DELETE the A record `@ -> 23.21.234.173`
   - DELETE the A record `@ -> 23.21.157.88`
   - ADD a new A record: Name `@`, Value = the IP from Vercel (likely `76.76.21.21`), Proxy status = **DNS only (gray cloud)**
   - UPDATE the www CNAME: change target to `cname.vercel-dns.com`, Proxy status = **DNS only (gray cloud)**
2. CRITICAL: Both records MUST have the orange cloud toggled OFF (gray cloud / DNS only). If Cloudflare proxy is on, Vercel SSL will break.

After completing all three steps, type "done" and Claude will run verification commands.
  </action>
  <how-to-verify>
Claude runs these verification commands after user reports completion:

```bash
# 1. Check DNS resolves to Vercel IP
```

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Get GSC token, replace placeholder, deploy, verify GSC, submit sitemap</name>
  <action>
This task has a human portion (getting the token, final GSC verification) and a Claude-automated portion (code change, push, deploy verification).

**Step 1 -- Get GSC verification token (user):**
1. Go to https://search.google.com/search-console
2. Click "Add property"
3. Select "URL prefix" (NOT "Domain")
4. Enter: `https://joescannell.com`
5. Google shows verification methods. Select "HTML tag"
6. Google displays a meta tag. Copy ONLY the content value (the token string, e.g., `abc123XYZrealToken`). Do NOT copy the full HTML tag.
7. Provide the token to Claude

**Step 2 -- Replace placeholder and deploy (Claude automates):**
Claude will:
1. Open `app/layout.tsx`
2. Replace `'GSC_VERIFICATION_TOKEN_PLACEHOLDER'` with the real token
3. Commit with message `feat(02-01): replace GSC verification placeholder with real token`
4. Push to GitHub (triggers Vercel auto-deploy)
5. Wait for Vercel deployment to complete
6. Verify the token appears in the live page source:
   ```bash
   curl -s https://joescannell.com | grep google-site-verification
   # Expected: <meta name="google-site-verification" content="REAL_TOKEN_HERE"/>
   ```

**Step 3 -- Verify in GSC and submit sitemap (user):**
1. Return to Google Search Console
2. Click "Verify" for the joescannell.com property
3. GSC should find the meta tag and confirm verification
4. Go to Indexing > Sitemaps in the left nav
5. Enter `sitemap.xml` in the "Add a new sitemap" field
6. Click "Submit"
7. GSC acknowledges the sitemap (status may show "Pending" initially, that is normal)

After completing GSC verification and sitemap submission, type "done".
